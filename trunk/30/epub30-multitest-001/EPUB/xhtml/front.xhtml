This is one of the documents in the EPUB3 Compliance test suite.

This particular document focuses on human-evaluated binary (pass/fail) tests in a reflowable context. Tests for fixed-layout content and other individual tests that require a dedicated epub file are available in additional sibling documents.

The following conventions are used throughout the document:

1. Locating a test

Tests that pertain to required Reading System functionality are preceded by a "locator box" that looks like this: Test-001

Tests that pertain to optional Reading System functionality are preceded by a "locator box" that looks like this: Test-002

2. Performing the test
Following each "locator box" is a description of what the given test is about, and then the actual test statement, which can always be evaluated to true or false. These statements typically have the form "If [some condition], the test passes".
